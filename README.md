# JavaMessengerProject
Данный проект представляет собой многофункциональное приложение для общения, обмена сообщениями, и социальных взаимодействий. Ниже приведено описание основных функций и сущностей этого мессенджера:

1) Users (Пользователи):

Модель Users представляет пользователей приложения.
Она содержит информацию о пользователях, такую как имя, адрес электронной почты, пароль и другие учетные данные.
Пользователи могут регистрироваться, аутентифицироваться и управлять своими учетными данными.
Могут отправлять запросы на дружбу.

2) UserProfile (Профиль пользователя):

Модель UserProfile используется для отображения профилей пользователей.
Он содержит информацию о пользователе, такую как аватар, биография и другие персональные данные.
Пользователи могут просматривать свой собственный профиль и профили других пользователей.

3) Chats (Чаты):

Модель Chats предназначена для хранения истории переписки.
В чатах пользователи могут обмениваться текстовыми сообщениями и файлами.
Мессенджер поддерживает как одиночные чаты, так и групповые чаты.

4) Messages (Сообщения):

Модель Messages представляет текстовые и файловые сообщения, отправляемые пользователями.
Сообщения могут содержать текст, изображения и другие медиафайлы.
Пользователи могут отправлять, просматривать и удалять сообщения.

5) FriendsList (Список друзей):

Модель FriendsList содержит информацию о друзьях пользователя.
Пользователи могут добавлять других пользователей в свой список друзей и принимать запросы на дружбу.

6) Posts (Посты):

Модель Posts предназначена для создания, оценивания и комментирования постов.
Пользователи могут создавать текстовые посты, прикреплять изображения и видео, а также ставить лайки и писать комментарии к постам других пользователей.

7) Comments (Комментарии):

Модель Comments используется для хранения комментариев к постам и другим медиа-контенту.
Пользователи могут добавлять комментарии к постам и обсуждать их с другими пользователями.


Проект мессенджера сочетает в себе функциональность мессенджера, социальной сети и платформы для обмена контентом. Пользователи могут общаться, делиться содержанием, заводить друзей, и создавать ленту новостей с помощью постов и комментариев. Это приложение обеспечивает разнообразный опыт общения и социальных взаимодействий для своих пользователей.

## Описание функциональности:
 ### 1.Chat Controller
1) ChatController является основным контроллером, ответственным за управление операциями, связанными с чатами, в приложении. Он предоставляет конечные точки для различных действий, связанных с чатами, включая создание, обновление и удаление чатов, получение списков чатов, сообщений и многое другое. Этот контроллер в первую очередь служит для обработки функциональности чатов в приложении.

Доступные пути:

GET /chat: Получить список чатов.

    Доступно для: Только администраторов.
GET /chat_list_of_user/{username}: Получить список чатов для конкретного пользователя.

    Доступно для: Администраторов и пользователя, указанного в параметре пути {username}.
GET /{id}: Получить информацию о конкретном чате по его идентификатору.

    Доступно для: Только администраторов.
GET /messages/{id}: Получить сообщения для конкретного чата по его идентификатору.

    Доступно для: Администраторов и авторизованных пользователей с доступом к чату.
GET /{chatId}/find_message/{messageId}: Найти конкретное сообщение в чате по его идентификатору.

    Доступно для: Администраторов и авторизованных пользователей с доступом к чату.
POST /: Создать новый чат.

    Доступно для: Только администраторов.
PUT /: Обновить данные чата.

    Доступно для: Только администраторов.
PUT /update_name/{id}: Обновить название чата.

    Доступно для: Только администраторов.
PUT /update_description/{id}: Обновить описание чата.

    Доступно для: Только администраторов.
DELETE /{id}: Удалить чат по его идентификатору.

    Доступно для: Только администраторов.
DELETE /clear_messages/{chatId}: Очистить все сообщения в чате по его идентификатору.

    Доступно для: Только администраторов.
DELETE /{chatId}/delete_message/{messageId}: Удалить конкретное сообщение в чате по его идентификатору.

    Доступно для: Только администраторов.
DELETE /{chatId}/delete_user/{userId}: Удалить пользователя из чата по его идентификатору.

    Доступно для: Только администраторов.

### 2.User Controller
2) UserController - это контроллер, отвечающий за выполнение всех операций, связанных с пользователями в приложении. Он предоставляет конечные точки для управления пользователями, включая получение списка пользователей, поиск пользователей по имени и фамилии, получение информации о конкретном пользователе, создание, обновление и удаление пользователей.

Доступные пути:

GET /user: Получить список пользователей.

    Доступно для: Только администраторов.
GET /user/last: Получить пользователя по фамилии.

    Доступно для: Всех пользователей.
GET /user/first: Получить пользователя по имени.

    Доступно для: Всех пользователей.
GET /user/{id}: Получить информацию о конкретном пользователе по его идентификатору.

    Доступно для: Только администраторов и авторизованных пользователей, чей идентификатор совпадает с {id}.
POST /user: Создать нового пользователя.

    Доступно для: Всех пользователей (возможно, стоит рассмотреть ограничение только для администраторов).
PUT /user: Обновить данные пользователя.

    Доступно для: Только администраторов и пользователя, чей идентификатор совпадает с идентификатором входа.
DELETE /user/{id}: Удалить пользователя по его идентификатору.

    Доступно для: Только администраторов и пользователя, чей идентификатор совпадает с {id}.
Контроллер UserController позволяет администраторам управлять пользователями в системе, а также пользователям управлять своими собственными данными (в зависимости от реализации приложения).

### 3.UserProfileController
3) UserProfileController - это контроллер, отвечающий за выполнение всех операций, связанных с профилем пользователя в приложении. Он предоставляет конечные точки для управления профилем пользователя, включая получение списка профилей пользователей, получение информации о конкретном профиле пользователя, обновление профиля, загрузку изображения профиля и удаление профиля пользователя.

Доступные пути:

GET /user_profile: Получить список профилей пользователей.

    Доступно для: Только администраторов.
GET /user_profile/{id}: Получить информацию о конкретном профиле пользователя по его идентификатору.

    Доступно для: Всех пользователей.
GET /user_profile/friend/{id}: Получить профиль другого пользователя по его идентификатору.

    Доступно для: Всех пользователей.
POST /user_profile: Создать новый профиль пользователя.

    Доступно для: Всех пользователей (возможно, стоит рассмотреть ограничение только для администраторов).
PUT /user_profile: Обновить данные профиля пользователя.

    Доступно для: Только администраторов и пользователя, чей идентификатор совпадает с идентификатором входа.
PUT /user_profile/upload/{id}: Загрузить изображение профиля пользователя.

    Доступно для: Только администраторов и пользователя, чей идентификатор совпадает с {id}.
DELETE /user_profile/{id}: Удалить профиль пользователя по его идентификатору.

    Доступно для: Только администраторов и пользователя, чей идентификатор совпадает с {id}.
Контроллер UserProfileController предоставляет возможность пользователям управлять своими профилями, включая загрузку изображения профиля, а также администраторам управлять профилями пользователей в системе.
